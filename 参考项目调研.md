### 起步参考

主参考 **ADHDAlarms**（覆盖 AlarmManager / 三种 Presentation / 授权 / SwiftData 持久化 / 自定义 UI，直接照抄可跑通），场景灵感看 **Alare**（防睡过头 + 唤醒动作交互），最小骨架看 BleepingSwift。App Intents 缺失可补看 fmorandin/focused-time。

### "闹钟组联动"——无现成，需自研

现有项目**没有任何一个实现"关一个关全组"**。alarmee 的 groupKey、alarmo 的 sensor group 都只是"展示分组"而非"联动开关"。需自建：

1. 数据模型加 `groupID` 字段；
2. 关闭回调里查同组全部 alarm，逐个调 `AlarmManager.cancel(id:)`；
3. **关键坑**——批量操作要**串行 / 加节流**，避免并发取消丢失（参考 home-assistant Sonos issue）；
4. App Intents / 通知动作触发关闭时也要走同一分组逻辑，避免多入口重复代码（natsu1211 README 专门提到这点）。

### 值得借鉴的功能点

数学题(MathAlarm)、摇一摇(ShakeAlarmClock)、Alare 的挥动/扫码/鼓点动作、渐进音量+贪睡(Fossify/Simple-Clock)、睡眠周期唤醒窗口(Sleepwell)、通勤动态唤醒(smart-alarm/ios)、Core ML 声音识别(SleepWithAI)。

### 支付宝/微信解析能否移植 Swift

**没有 Swift 现成实现**，生态全在 Python/Go。可直接照搬的是**逻辑而非代码**：

- **字段映射/规则**：抄 `double-entry-generator` 的 YAML 规则（按 category/time/peer 命中目标账户）和 `china_bean_importers` 的 `narration/payee_keywords + priority + match_logic(OR/AND)`。
- **去重（最有价值）**：DEG discussion #147/#162 给出成熟策略——支付宝/微信与银行卡同一笔重复时，**保留信息量大的支付工具侧**（有订单号/商品名），忽略银行卡侧；卡间转账才两边都记。Aubade 可用「订单号+金额+时间」做去重键。
- **Swift 落地层**：CSV 用 `swiftcsv/SwiftCSV`，微信 2025+ xlsx 用 `CoreOffice/CoreXLSX`（纯 Swift）。解析规则用 Swift 重写即可。

### 数据模型最值得借鉴

**ezBookkeeping**（体量最贴近个人 App）：二级账户 + 二级分类，交易类型分收入/支出/转账。概念上层参考 **Beancount 复式模型**（account/posting），Firefly III 的「账户-交易-预算-规则-标签」是成熟范式。

Aubade 建议模型：`Account / Category(两级) / Transaction(含 type、rawSource、orderId 去重键) / Budget`。

### URL Scheme / 快捷指令自动记账

**AutoLedger 是唯一可参考的开源实现**：用 `AppIntents` + `ForegroundContinuableIntent` + Share Extension + Deep link Router，并附一个可安装的快捷指令。URL Scheme 回传可参考 `phimage/CallbackURLKit`（x-callback-url）。iCost / 蜜蜂那类闭源，无开源版；AutoLedger 最接近。

## 三、Aubade 落地建议总结

- **自己写**：三条自动记账链路（短信→Shortcuts→URL Scheme、OCR 回传、账单导入去重）+ 闹钟组联动——这是 Aubade 的特色，无现成可套。
- **抄现成**：AlarmKit 骨架抄 ADHDAlarms；记账架构抄 AutoLedger；解析/去重逻辑抄 double-entry-generator；数据模型抄 ezBookkeeping；UI/预算/Widget 抄 Dime。
- **直接用库**：SwiftCSV + CoreXLSX（文件解析）、Swift Charts（报表）、CallbackURLKit（x-callback-url）。
- **优先精读**：`ADHDAlarms`（闹钟起步）和 `AutoLedger`（一个仓库覆盖记账三条链路）。

# Aubade v1 产品需求方案

## 给用户看的摘要

Aubade 是一个**私人自用的 iOS 记账 App**，不上架 App Store，只装在你自己的 iPhone 上。它要解决的核心问题是：**让记账这件事尽量不用你动手**——收到收据/账单截图能识别记一笔，随口说一句也能记一笔，把工行短信复制进来能记一笔，也能手动补一笔。记完的账由 AI 自动归类并**直接入账**，你可以设周/月预算看花超没超，还能看到"我所有钱现在还剩多少"。

**方案概述**：v1 聚焦"多入口记账 + 智能分类 + 预算统计 + 剩余金额"四件事，数据全部存在手机本地（十年也才几 MB）。AI 能力接 DeepSeek，但 DeepSeek 的 API 只处理文本——所以截图记账是先用 iPhone 本地的文字识别（iOS Vision）把图里的字抽出来，再把**文本**交给 DeepSeek 解析，图片本身不外传；语音记账同理，先用 iPhone 本地语音识别转成文字再解析。截图记账的主入口是 **iOS 快捷指令**：你在任意付款页/账单页随手一截，快捷指令把图片发给 Aubade，App 在**后台**完成识别并直接入账，只弹一条本地通知告诉你结果，不打断你正在做的事。所有记账**识别/填写完直接入账**，记错了可随时在账单列表里改。

要点如下：
1. **四种记账入口**：截图识别（iOS 快捷指令随手截图后台入账，也可在 App 内从相册选图）、语音输入、文本识别（粘贴工行短信或任意含金额文字）、手动输入。
2. **识别后直接入账**，不设"待确认队列"；记错了在列表里改即可。
3. **剩余金额 = 你所有账户的总余额**：首次使用时录入一个初始总额，之后每笔收入加、每笔支出减，滚动显示当前总额。
4. **分类体系**：支出分衣、食、住、行、玩、其他；收入分工作（工资）、其他收入。

---

## 产品定位

- 单用户、私人自用的 iOS 记账工具，无多人协作、无账号体系、无社交。
- 核心价值是**降低记账的手动成本**：一段截图/语音/短信文本能一步变成结构化账单，不用逐字手输。
- 数据主权完全归用户：纯本地存储，除把识别后的文本发给 DeepSeek 做解析/分类外，不联网、不上云。

## 产品目标

1. 让记一笔账尽量快：截图（快捷指令随手截图后台入账）、语音、文本识别三种识别类方式都能把一段原始信息一步变成结构化账单，也支持手动补录。
2. 记入的账单自动归类并直接入账，用户基本不用手动选分类、也不用逐笔确认。
3. 用户能随时看清：本周/本月花了多少、有没有超预算、所有钱加起来还剩多少。
4. 记错、认错都能快速在账单列表里改正。

## 用户视角使用流程

### 主流程 A：截图识别记账（iOS 快捷指令随手截图）

1. 用户预先在"快捷指令"里配置一个动作：截屏 → 把图片发给 Aubade。之后在任意场景（支付宝/微信/银行的付款结果页、账单详情页等）付完款，用户主动触发这个快捷指令（如轻点背面、从分享菜单选、或语音唤起）。
2. 快捷指令截取当前画面并把图片交给 Aubade。Aubade 在**后台**先用 iPhone 本地的文字识别（iOS Vision）把图里的文字抽出来，再把这段文本交给 DeepSeek 解析出金额、收支方向、商户、时间等信息，并自动判定分类（图片本身不外传）。
3. Aubade **直接把这笔账记入账本**，并弹出一条**本地通知**告知结果（如"已记一笔 · 支出 ¥35 · 餐饮"），不强制打断用户当前操作。
4. 用户点通知可打开 Aubade 查看这笔账的结果卡片、当场修改；不点也已经记好。
5. 完成状态：账本里多了一笔已分类的账单，统计和剩余金额同步更新；如有识别错误，用户可在列表里改。
6. **备选路径**：用户也可以在 App 内点"截图识别 → 从相册选图"，选一张已保存的付款截图/账单图来识别，流程同上（本地 OCR → DeepSeek → 直接入账）。适合当时没触发快捷指令、事后想补录的情况。

> 说明：这里的快捷指令由用户**主动触发**、手机处于**解锁在用**状态，因此能可靠把图片交给 App；这与"工行短信自动记账"不同——那种要在**锁屏**时被动触发并唤起 App，iOS 不支持，故 v1 不做（见"v1 不包含"）。

### 主流程 B：语音记账

1. 用户点"语音记账"，说一句话，例如"今天中午吃饭花了 35 块"。
2. Aubade 用 iPhone 本地的语音识别（iOS Speech）把语音转成文字，再用 DeepSeek 解析成结构化账单（金额、方向、分类、备注）。
3. Aubade 直接入账并提示；用户如需修改可在列表里改。

### 主流程 C：文本识别记账

1. 用户复制一段含金额的文字——最常见是工行的到账/扣款短信，也可以是任意支付结果文字、聊天里的一句话等。
2. 打开 Aubade，点"文本识别"，把文字粘进去（若剪贴板已有内容，可一键带入）。
3. Aubade 用 DeepSeek 解析出金额、收支方向、时间、对方/商户、卡尾号（若有），自动判定分类并直接入账。
4. 完成状态：账单入账并归类，统计与剩余金额更新。

### 主流程 D：手动记账

1. 用户点"手动记账"，填写金额、收支方向、分类、日期、备注。
2. 保存后入账。

### 辅助流程 E：查看统计与设置预算

1. 用户打开"统计"页，看到本周/本月总支出、按分类的占比、预算完成进度。
2. 用户可设置**周预算**和**月预算**；当支出接近或超过预算时，页面给出明显的进度/超支提示。

### 辅助流程 F：查看剩余金额

1. 用户打开"剩余金额"页，看到当前**所有账户合计的可用总额**（初始总额基础上，收入累加、支出扣减后的滚动结果）。
2. 首次使用时，用户录入一个初始总额作为计算起点。

### 辅助流程 G：账单管理

1. 用户在账单列表按时间浏览所有账单，可编辑、删除、修正分类。
2. 对识别有误的账单，用户能快速改正，改正后统计与剩余金额同步更新。

## 需要维护的数据

- **账单（Transaction）**：金额、收支方向（支出/收入）、发生时间、分类、对方/商户（可空）、备注、卡尾号（可空，仅记录）、来源（截图/语音/短信/手动）、原始输入文本或截图引用（用于回溯与改正）。其中"截图"来源可进一步区分是快捷指令送入还是 App 内相册选图（仅作记录，便于回溯）。
- **分类（Category）**：名称、收支属性、（可选）图标/颜色；系统预置一套，用户可增删改。
- **预算（Budget）**：周期类型（周/月）、金额；周预算与月预算可同时存在。
- **余额基线（BalanceBaseline）**：用户录入的初始总额及其录入时间点，作为剩余金额的滚动计算起点。
- **应用设置**：DeepSeek API Key、预算周期起止规则、记账结果通知开关等。

## 数据交付物

- **账单列表**：按时间倒序展示，支持按分类/时间范围筛选，可编辑、删除。
- **统计视图**：本周/本月总支出、总收入、分类占比、预算进度与超支提示。
- **剩余金额视图**：当前所有账户合计的可用总额。

## 业务规则

1. **收支方向识别**：截图/语音/短信解析时，必须区分收入与支出（工行短信含"支出/消费"与"收入/入账"等语义）。
2. **金额与时间归一**：金额统一为数值（元），时间统一为可比较的日期时间；识别不到时间时默认取当前时间。
3. **直接入账**：所有来源（含 AI 识别）的记账，识别/填写完成后**直接记为正式账单**，不经过待确认队列；识别错误由用户在账单列表里事后修改。
4. **快捷指令截图入账**：截图记账的主入口是 iOS 快捷指令，由用户主动触发把图片交给 App；App 在后台完成"本地 OCR → DeepSeek 解析 → 直接入账"，并弹一条本地通知告知结果，用户点通知可查看/修改这笔账。App 内"从相册选图"是等效的备选入口。
5. **记账结果通知**：后台入账成功后发本地通知（如"已记一笔 · 支出 ¥35 · 餐饮"）；识别失败或未配置 Key 时，通知给出对应提示并引导用户打开 App 处理。通知可在设置里关闭。
6. **智能分类**：系统依据商户名、备注、原始文本等信息自动推断分类；用户可随时改正，改正结果不影响其他账单。
7. **分类体系（预置，可增删改）**：
   - 支出：衣、食、住、行、玩、其他。
   - 收入：工作（工资类收入）、其他收入。
   - 归类约定：医疗、话费、数码等不单列的支出归"其他"；红包、退款、转账进账等不单列的收入归"其他收入"。
8. **预算周期**：周预算默认按自然周（周一至周日）、月预算按自然月统计；支出累计到对应周期，用于计算进度。
9. **超预算提示**：支出达到预算的设定比例（如 80%）时提示接近，超过 100% 时明显标红，仅提示不阻止记账。
10. **剩余金额计算**：剩余金额 = 初始总额 + 基线时间点之后的收入合计 − 之后的支出合计；仅在有基线时展示。
11. **识别容错**：任何识别结果都可被用户修改；识别失败时降级为手动填写，且已采集的原始文本/截图不丢失，用户可基于它继续补录。为支持"失败后补录/用原图重试"，快捷指令送入的原图**临时留存在本机**，成功入账或用户放弃后即可清理；这不同于"附件长期图库管理"（v1 不做，见"v1 不包含"）。
12. **本地优先**：截图的文字识别（iOS Vision）、语音转文字（iOS Speech）均使用 iPhone**系统本地能力**完成，图片/语音不外传；只有识别后的**文本**会发给 DeepSeek 做分类与结构化解析。除此之外数据不出本机；DeepSeek Key 由用户在 App 内自行填写与保存。

## 验收点

1. 通过 iOS 快捷指令把一张付款截图发给 Aubade，App 在后台先本地识别出文字、再解析出金额与方向，直接生成一笔已入账、带分类的账单，并弹出一条含金额/分类的本地通知。
2. 在 App 内点"截图识别 → 从相册选图"选一张付款截图，同样能本地识别、解析并直接入账。
3. 说一句"打车花了 20"，能直接生成一笔金额 20、方向为支出、分类为"行"的已入账账单。
4. 把一条工行消费短信（或其他含金额的文字）粘贴进"文本识别"，能解析出方向/金额/时间并直接入账。
5. 手动新增一笔账单可成功保存并出现在列表中。
6. 新入账的账单带有一个自动分类，且用户可修改分类，改后统计同步更新。
7. 统计页能正确显示本周与本月的支出合计与分类占比。
8. 设置周预算/月预算后，统计页能显示进度，超支时有明显提示。
9. 录入初始总额后，剩余金额页显示的数值 = 初始总额 + 后续收入 − 后续支出。
10. 账单列表支持编辑与删除，操作后统计与剩余金额同步更新。
11. 未配置 DeepSeek Key 时，识别类功能给出明确提示（含快捷指令后台入账时以通知形式提示），手动记账仍可正常使用。
12. 账单列表可按分类和时间范围筛选，结果与筛选条件一致。
13. 当一条截图/语音/短信无法解析出金额时，App 不报错丢失，而是保留原始文本让用户手动补全后入账；若是快捷指令后台截图解析失败，则弹一条失败通知，点开后原文/原图仍在，可据此补录。

## v1 不包含

- 闹钟功能。
- 工行短信的**自动**记账（收到短信自动触发并唤起 App）：iOS 不支持锁屏时被动唤起第三方 App，故只做手动粘贴短信文本，不做短信自动触发。（注：截图记账用的快捷指令由用户**主动触发**，不受此限制，见主流程 A。）
- 支付宝、微信的记账（无个人开发者接口，聚焦工行）。
- 待确认/审核队列：识别后直接入账，不做逐笔确认流程。
- 多银行/多账户的**独立账户体系**：只维护一个合计总余额，卡尾号仅作记录字段，不分账户统计。
- 支付宝/微信月度账单 CSV/xlsx 导入。
- iCloud 同步、云端后端、多设备同步。
- 数据导出/报表导出、图表深度分析（趋势预测等）。
- 多币种、汇率。
- 账单附件的长期图库管理。

## 后续澄清方向

1. **预算起止规则**：周预算按周一起算是否符合习惯；是否需要"跨周期结转"。
2. **超预算阈值**：接近提示的比例（默认 80%）是否合适，是否需要可配置。
3. **分类可维护程度**：v1 是否需要让用户在设置里增删改分类，还是先用固定的预置分类。
4. **文本识别的便捷度**：是否需要在 App 内提供"读取剪贴板一键带入"，还是让用户手动粘贴即可。
5. **快捷指令的接入形态**：App 对外暴露记账动作用哪种技术（App Intents / 分享扩展 / URL Scheme），以及快捷指令与 App 之间传图的具体方式——留待技术基线阶段定，PRD 层面只约定"用户主动触发截图 → 后台入账 → 通知"的行为。
6. **后台识别失败的通知措辞与重试**：识别失败/超时/无网时的通知怎么提示、是否提供"用原图重试"入口。

## 已确认决策

- 平台：iOS，私人自用，**不上架 App Store**；使用 Mac + Xcode 开发。
- 记账入口共四种：**截图识别（iOS 快捷指令随手截图后台入账，也可 App 内从相册选图）、语音输入、文本识别（粘贴短信或任意含金额文字）、手动输入**。
- 截图记账的主入口是 **iOS 快捷指令**：用户主动触发把截图交给 App，App 在**后台**完成识别并直接入账，弹本地通知告知结果；不点通知也已记好，点通知可查看/修改。
- 智能能力接入 **DeepSeek**（负责**文本的分类与结构化解析**）；截图的文字识别用 iOS Vision、语音转文字用 iOS Speech，均为**本机系统能力**，图片/语音不外传。**DeepSeek API Key 由用户在 App 内直接填写**。
- 识别/填写完成后**直接入账**，不设待确认队列；事后可在账单列表修改。
- **剩余金额 = 所有账户合计总余额**：首次录入初始总额，之后随每笔收支滚动加减，不区分具体账户。
- 分类体系：支出 衣/食/住/行/玩/其他，收入 工作（工资）/其他收入。
- 数据存储：**纯本地 SwiftData**，不做云端/后端/iCloud 同步。
- 短信来源聚焦**工商银行（95588）**。
- 需支持**周预算与月预算**，以及**剩余金额**页面。

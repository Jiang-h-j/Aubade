# TRD 02 - D7 双重扣减提示（两处录入入口）

## 给用户看的摘要

切片 01 改成全量口径后，有一个你已经确认接受的边界：如果你在录初始总额时填的净值**本来就已经扣过**某笔早期消费，而那笔消费又还作为账单存在，它会被扣两次。这一片就是在**你录初始总额的两个地方**——首次引导第一步、我的页「调整初始总额」——各写一句短提示：**填当前净值就好，别再补录初始总额之前的历史账，否则会重复扣减。** 纯改文案、不动任何录入逻辑，也不做去重对账。

## 本 TRD 负责什么

- 在我的页 `InitialBalanceSheet` 录入处（footer）加/改写 D7 双重扣减提示。
- 在首次引导 `OnboardingView` 步①说明文案上强化同一提示。
- 两处提示措辞简短一致、纯展示、不阻塞录入流程。
- 覆盖 PRD 验收标准 4（提示可见）；确认验收 5（未录基线分支不回归）不受影响。

```swift
    Text("之后每记一笔收支，剩余总额会自动加减。")            // :327  ← 在此追加 D7 提示
}
```

- `parsedAmount`（`:310-316`）、`onSave`（`:339`）、保存按钮 disable（`:343`）、`onAppear` 预填（`:346-350`）**全不动**。

**落点二：首次引导步①**（`Aubade/Features/Onboarding/OnboardingView.swift:73-112`，`balanceStep`）。当前说明文案：

```swift
Text("你现在大约有多少钱？")                                // :76
    .font(.title3.weight(.semibold))
Text("所有账户加起来的合计，作为剩余金额的起点。之后每记一笔收支会自动加减。可以先跳过，稍后在「我的」里设置。")  // :78  ← 在此强化 D7
```

- `parsedAmount`（定义 `:29`，使用点 `:94`）、`setBalanceBaseline`（`:95`）、「下一步 / 先跳过」按钮（`:93-109`）**全不动**。

两处文案表述已高度相似（都含「之后每记一笔收支会自动加减」），D7 提示应在这句之后自然衔接，措辞一致。

## 设计方案

**统一提示措辞**（两处共用同一句核心，简短、口语、不吓人）：

> 填当前净值就好，别补录初始总额之前的旧账，否则会被重复扣减。

**落点一（我的页 footer）**：把现有 footer 从单句扩成两句，第二句是 D7 提示。footer 天然是浅色小字说明位，不阻塞输入：

```swift
} footer: {
    Text("之后每记一笔收支，剩余总额会自动加减。填当前净值就好，别补录初始总额之前的旧账，否则会被重复扣减。")
}
```

**落点二（引导步①说明）**：在现有说明句尾接上 D7 提示（与落点一同一核心句），保持整段仍是浅色副标题：

```swift
Text("所有账户加起来的合计，作为剩余金额的起点。之后每记一笔收支会自动加减。填当前净值就好，别补录初始总额之前的旧账，否则会被重复扣减。可以先跳过，稍后在「我的」里设置。")
```

（引导句已含「可以先跳过…」尾句，D7 提示插在「自动加减」之后、「可以先跳过」之前，语序更顺。）

两处均为 `Text` 文案字符串替换，无新增视图、无新增状态、无逻辑分支。

## 修改点

- **改** `Aubade/Features/AppShell/RootTabView.swift`：`InitialBalanceSheet` 的 footer `Text`（现 `:327`）字符串追加 D7 提示句。仅改这一个字符串字面量。
- **改** `Aubade/Features/Onboarding/OnboardingView.swift`：`balanceStep` 说明 `Text`（现 `:78`）字符串插入 D7 提示句。仅改这一个字符串字面量。

## 验证点

1. **目视（验收 4）**：我的页「我的」→「调整初始总额」sheet 打开，输入框下方 footer 可见 D7 双重扣减提示；首次引导（全新安装或重置 onboarding 标志）步①录初始总额页，说明文案含同一 D7 提示。
2. **文案一致性**：两处提示核心句表述一致（都传达「填当前净值、勿补录旧账、否则双扣」）。
3. **不阻塞（回归）**：提示为纯展示，录入 → 保存 → 写库流程不变；保存按钮 disable 条件、跳过按钮行为均不受影响。
4. **编译**：全 target 编译通过（folder-based 项目，仅改字符串不涉新增文件/pbxproj）。
5. **验收 5 确认**：本片不碰 `remaining`/`baseline == nil` 分支，未录初始总额仍显示「未设置/—」，无回归。

## 不做什么

- 不做去重 / 对账 / 净值拆解等任何算法逻辑（D7 是文案规避，非代码消除；用户已确认接受该边界）。
- 不改 `parsedAmount`、`setBalanceBaseline`、`onSave`、保存/跳过按钮的 disable 与行为。
- 不新增视图组件、状态、弹窗或阻塞式确认；仅替换既有 `Text` 字符串。
- 不改 `InitialBalanceSheet` 的 `private` 可见性、不抽公共组件（两处各自内联改文案即可）。
- 不碰切片 01 的 `BalanceCalculator` 逻辑。
- 不碰 B02~B04 范围、不改数据模型、不动统计链路。

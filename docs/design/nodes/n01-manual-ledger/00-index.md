## 切片划分与理由

本节点拆 **3 个 TRD 切片**。与 N00 单切片不同：N01 是纯 UI 构建，界面面广，且带一条**对下游 N03~N06 的接口承诺**（可复用编辑组件），存在真实的"组件先于消费方"的实现顺序边界，满足拆多切片的触发条件。

拆片依据（每片可独立编译、可肉眼或单测验收，符合 DAG「节点内可分步验证」）：

- **切片 01（地基）**：4-Tab 主框架替换 N00 占位 `ContentView` + 分类展示映射 `CategoryStyle`。这两者是 02/03 的公共底座——框架不立起来后续视图无处挂载；分类"派生色 + emoji"被表单选择器、列表标签、编辑页共同消费（PRD §当前理解指出预置分类 color/icon 当前为 nil，须在端上兜底）。先落地基。
- **切片 02（核心复用件 + 写入闭环）**：可复用编辑组件 `TransactionEditor`（新建草稿 / 编辑已有双模式）+ 手动记账表单 + 记账 Tab。编辑组件是本节点对 N03~N06 最重要的接口承诺（识别结果卡片直接复用其字段编辑与保存），**必须先于切片 03 的"从列表进编辑"落地**，否则 03 无组件可用。本片打通"手动记一笔 → 入账 → 记账页即时反馈"。
- **切片 03（读取闭环）**：账单 Tab 流水列表（日期分组 / 彩色标签 / 方向色）+ 分类/时间筛选 + 详情编辑（复用 02 的组件）+ 删除二次确认。依赖 02 的编辑组件已就位，构成"看流水 → 改 → 删"闭环。

切片顺序 01 → 02 → 03 是硬依赖顺序：02 依赖 01 的框架与映射，03 依赖 02 的编辑组件。无逆向依赖、无环。

## 切片划分与理由

本节点只有 **一个 TRD 切片**。

N00 的六项交付物——Xcode 工程骨架、四个 SwiftData 模型、`ModelContainer` 封装点、预置分类首次装载、薄读写封装、验证手段——**彼此强耦合且不可独立验证**：没有工程就没有模型编译环境，没有模型就无法建容器，没有容器就无法装载预置与跑 CRUD。强行拆片只会制造"半个不能编译运行"的中间态，违背 DAG「每个节点都能独立跑通并验收」的原则。因此它们构成**一个可编译、可运行、可用单元测试验证的最小闭环**，收敛为单一切片。

无共享代码风险、无迁移风险、无"不可同时验证的实现边界"（全部是全新文件），不满足拆多切片的任何触发条件。

# 反馈批次 01 原型与交互设计 — 剩余总额 / 日期填充 / 最近记录删除 / 自定义分类 / 截图多笔 / UI 还原

## 给用户看的摘要

这份是**增量原型**：不重画整个 App，只把这批 6 条反馈里**新增/改动的交互**做成你能在浏览器里直接点的样子，接着上一版原型 `prototype/app/` 往上加。上一版已验收的记账/账单/统计/我的四页、识别结果卡、快捷指令通知都还在，本批只在它们身上补东西。

**请你在 demo 里重点点这几条新路径**（`prototype/app/index.html` 双击打开）：

1. **截图多笔（R5，改动最大）**：进「记账」→ 点「截图识别」→ 点最下面新增的 **`🧾 演示：选一张含多笔的账单截图`** → 看识别中转出**一张卡片列出 3 笔**，每笔能点进去改、右边 `✕` 能单独删（删到 0 笔自动关）→ 点「完成」→ 账单页出现这几笔。
2. **日期未识别提示（R2）**：底部演示控制台勾 **`识别不到日期`** → 走一次「文本识别」→ 结果卡的「时间」旁多了红色 **`日期未识别`** 标签 + 一行提示「已先按今天填，请确认」；账单列表里这笔也带「日期未识别」小字，你改过时间后标记消失。
3. **最近记录删除（R3）**：进「记账」→ 下半屏「最近记录」每条现在能**向左滑**露出「删除」→ 二次确认 → 删掉后剩余总额/统计同步变；和账单页删除同一套交互（账单页现在也能左滑删）。
4. **自定义分类（R4）**：进「我的」→「分类管理」现在不是只读标签了：预置分类标「预置·锁定」（点会提示不可改），底部 **`＋ 新增自定义分类`** 可建（选方向/名字/图标/颜色），自建的能点进去改名/换色/删除；删一个**已被账单用过**的分类会提示「N 笔转到其他」。
5. **剩余总额口径（R1）**：这条是算法 bug，demo 里的剩余数字**本来就是对的**（初始总额 + 全部收入 − 全部支出，不挑日期）——放这里是让你确认口径没歧义；真正要改的是 App 里的代码。

**这些反馈会影响下一步（技术基线）**：多笔识别的结果卡结构决定了解析契约要从「单笔」改成「多笔数组」；日期未识别的兜底策略（按今天填 + 高亮 vs 留空）决定识别层和结果卡字段；自定义分类的「删除已引用分类」策略决定 Store 层怎么处理外键。你现在觉得哪条交互多余、缺步骤或不对，**改原型比改代码便宜**，请在评审时一起提。

## 2. 交互原则（本批新增/强调）

- **识别即入账，多笔也一样**：多笔识别后**每笔都已是正式账**，多笔结果卡只是「看一眼 + 逐条可改可删」，与单笔结果卡同一心智，不引入「待确认队列」。
- **删除处处二次确认**：无论最近记录、账单列表还是多笔卡里的单笔删除，都走同一套二次确认弹窗，文案统一提示「剩余总额与统计会同步更新」。
- **识别不到的东西要让用户有感知**：日期识别不到时，不静默兜底——用红色标签 + 提示行让用户看见「这是猜的，请确认」。
- **预置与自定义边界清晰**：预置分类锁定（点击给出明确「不可修改」提示），自定义分类可自由增删改，视觉上分组、状态可辨。

## 3. 新交互页面骨架（低保真）

### 3.1 R5 截图多笔 · 识别结果卡

```
截图识别说明卡（v1 已有，本批加一个入口）
┌─────────────────────────────┐
│ 📷➜🌅  截图识别               │
│ （快捷指令主用法说明…v1 原样）  │
│ [▶︎ 演示：模拟收到快捷指令截图] │
│ ───────── 或 ─────────        │
│ [🖼 从相册选一张图识别]        │
│ [🧾 演示：选一张含多笔的账单截图]│ ← 本批新增入口
└─────────────────────────────┘
        ↓ 识别中…（本地读字 → DeepSeek 解析多笔）
┌─────────────────────────────┐
│ ✓ 已识别 3 笔                 │
│ 这张截图识别出 3 笔，已全部入账。│
│ 逐条看一眼，可单独改 / 删。     │
│ 支出合计 ¥188.50              │
│ ┌─────────────────────────┐ │
│ │🍜 星巴克   06-03 09:12  -38 ✕│ │ ← 点行进单笔编辑
│ │🍜 盒马鲜生 [日期未识别] -126.50 ✕│ │ ← 单笔 ✕ 删除
│ │🚗 滴滴出行 06-03 21:40  -24 ✕│ │
│ └─────────────────────────┘ │
│ ▸ 查看识别到的原始文本         │
│ [ 完成（3 笔已记） ]          │
└─────────────────────────────┘
```

- 点某笔 → 复用**单笔结果卡**编辑（改完回多笔卡刷新）。
- 单笔 `✕` → 二次确认「仅删这一笔，其余保留」；删到 0 笔自动关闭并 toast。
- 多笔中若某笔日期未识别，行内带 `日期未识别` 红字（R2 与 R5 叠加）。
- **失败/部分失败**（原型未逐一画，技术基线需定）：成功的入账、失败的提示、不产脏账——沿用单笔「识别失败不生成账单」原则。

### 3.2 R2 日期未识别 · 结果卡

```
┌─────────────────────────────┐
│ ✓ 已记一笔                    │
│ 金额   [ -256.00 ]            │
│ 方向   ( 支出 ● / 收入 ○ )    │
│ 分类   [ 其他 ▾ ]             │
│ 时间   [日期未识别] [2026-07-10 00:00]│ ← 红标签
│   没从截图/文本里读到消费日期，  │ ← 提示行（暖色）
│   已先按今天填，请确认或改成真实日期。│
│ 商户   [ 京东商城 ]           │
│ ...                          │
└─────────────────────────────┘
```

- 用户改过时间 → `日期未识别` 标记清除。
- 账单列表 / 最近记录该笔的副信息行带 `· 日期未识别` 小字，直到被确认。
- **兜底策略是「按今天填 + 高亮」还是「留空强制填」**：原型采用前者（更少打断），此为待确认项（见 §6）。

### 3.3 R3 最近记录 · 左滑删除

```
记账页下半屏「最近记录」
┌─────────────────────────────┐
│ 最近记录              全部 ›  │
│ ┌─────────────────────────┐ │
│ │🍜 楼下面馆 07-10 12:30 -35│←左滑│ │
│ │        ┈┈露出┈┈ [ 删除 ] │ │
│ │🚗 滴滴出行 07-10 09:05 -20│ │
│ └─────────────────────────┘ │
│          ← 左滑一条可删除      │
└─────────────────────────────┘
        ↓ 点「删除」
   二次确认弹窗「删除这笔账单？
   删除后无法恢复，剩余总额与统计会同步更新。」
```

- 未滑动时点行 = 进编辑（原行为不变）；已露出时点行 = 收起。
- 与账单页删除**同一套** `swipeRow` + 二次确认，账单列表也支持左滑删。
- 真机对应 SwiftUI `swipeActions`；原型用指针拖拽模拟。

### 3.4 R4 我的页 · 分类管理

```
我的页「分类管理」区块（原只读标签流 → 可管理列表）
┌─────────────────────────────┐
│ 分类管理                      │
│ ┌─────────────────────────┐ │
│ │ 支出分类                  │ │
│ │ 👕 衣            预置·锁定 │ │ ← 点提示"不可修改"
│ │ 🍜 食            预置·锁定 │ │
│ │ 🐾 宠物          编辑 ›   │ │ ← 自定义可编辑
│ │ 收入分类                  │ │
│ │ 💼 工作          预置·锁定 │ │
│ │ ＋ 新增自定义分类          │ │
│ └─────────────────────────┘ │
└─────────────────────────────┘
        ↓ 点「＋ 新增」/ 点自定义分类
┌─────────────────────────────┐
│ 新增自定义分类 / 编辑分类      │
│ 方向  ( 支出 ● / 收入 ○ )     │ ← 编辑态不可改方向
│ 名称  [ 宠物 ]  (≤6 字)       │
│ 图标  [🐾][🎓][💊][🎁]...     │ ← 网格单选
│ 颜色  [●][●][●][●]...         │ ← 网格单选
│ [ 创建分类 / 保存修改 ]        │
│ [ 删除这个分类 ]（仅编辑态）    │
└─────────────────────────────┘
```

- **预置保护**：预置分类点击不进编辑，toast「预置分类不可修改」；原型默认锁全部字段（名/图标/色），是否放开预置图标色为待确认项。
- **删除已引用分类**：若该分类被 N 笔账单引用，确认框提示「有 N 笔账单会转到『其他』」，确认后账单 `cat` 转为「其他」再删分类。此为原型采用策略，待你拍板（禁止删 / 转其他 / 其他方案，见 §6）。
- **自动分类**：DeepSeek 识别出的分类名匹配到库中分类（含自定义）即自动归类，无独立 UI——加了自定义分类后自然生效。是否把自定义分类塞进 DeepSeek 的分类清单 prompt，为待确认项。

## 5. 状态与异常（本批新增，接上 v1 §5）

| 场景 | 状态呈现 | 恢复路径 |
|---|---|---|
| 截图识别出多笔 | 多笔结果卡，全部已入账，逐条可改/删 | 逐条确认或「完成」 |
| 多笔中删到 0 笔 | 自动关闭结果卡 + toast「已删除」 | 账单页可见剩余（若有） |
| 识别不到消费日期 | 结果卡「日期未识别」红标签 + 兜底提示行；列表带小字标记 | 用户改时间后标记清除 |
| 删除最近记录/账单（左滑） | 左滑露出「删除」→ 二次确认 | 取消/确认；确认后数字同步 |
| 点预置分类 | toast「预置分类不可修改」，不进编辑 | — |
| 删除被引用的自定义分类 | 确认框提示「N 笔转到『其他』」 | 取消/确认转移 |
| 新增分类重名（同方向） | toast「该方向已有同名分类」，不创建 | 改名重试 |

## 6. 待确认（评审时请拍板）

1. **R2 日期兜底**：识别不到日期时，「按今天填 + 红色高亮提示」（原型现状）够不够？还是要「留空 + 强制用户填」？
2. **R4 删除已引用分类**：原型采用「转到『其他』」。你要「禁止删（先改账单）」还是「转其他」还是别的？
3. **R4 预置分类**：预置分类是否允许改图标/颜色（名称肯定锁）？原型现在锁全部。
4. **R4 自动分类 prompt**：是否把用户自定义分类名塞进 DeepSeek 的分类清单，让 AI 更可能命中自定义分类？
5. **R5 多笔覆盖入口**：多笔只做「截图」，还是「文本识别」也支持多笔？后台快捷指令截图（N06）要不要多笔（通知怎么呈现「已记 3 笔」）？
6. **R5 部分失败**：一张图里部分笔识别失败时，成功入账 + 失败提示的具体呈现（原型未细化）。
7. **R6 还原范围**：核心三页先行，还是全部页面一次到位？暗色模式做不做？

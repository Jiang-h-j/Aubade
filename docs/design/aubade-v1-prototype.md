# Aubade v1 原型与交互设计

## 给用户看的摘要

这份文档把 PRD 里的记账流程，变成你可以**在浏览器里直接点**的低保真原型（无需 Xcode）。它不是最终 App 的样子，也不是生产代码——只是让你在写技术方案前，先把"每个功能怎么点、点完看到什么、出错怎么办"过一遍。

**请你重点体验这几条路径**（demo 里都能点通）：
1. **记一笔账的四种方式**：进"记账"Tab（打开 App 默认就在这里）→ 分别试截图、语音、文本识别、手动——重点看识别完弹出的**结果卡片**，确认卡片上该有哪些字段、能不能当场改顺手。
2. **账单列表**：首页就是流水，试试按分类/时间筛选、点一笔进去改分类或删除。
3. **统计的时间维度**：进"统计"Tab，顶部用「日/周/月/年」切粒度、用「‹ ›」翻时间——切到"日"能选某天看当天流水，切到"月/年"看该期的趋势和分类占比；去"我的"设一个月预算，回来看**超支时的标红提示**长什么样。
4. **剩余总额**：首次使用引导你录一个初始总额，之后每记一笔看顶部数字怎么变。

**哪些反馈会影响下一步（技术基线）**：
- 结果卡片的字段和"当场改"的交互，决定了识别结果的数据结构和编辑组件设计。
- 统计页要展示的维度（周/月切换、分类占比形式），决定统计聚合怎么算、要不要缓存。
- 你如果觉得某个流程步骤多余或缺失，现在提最省事——改文档比改代码便宜得多。

## 1. 设计范围

覆盖 PRD 的全部 v1 功能：

- 四种记账入口：截图识别（iOS 快捷指令随手截图后台入账为主、App 内相册选图为备选）、语音记账、文本识别（粘贴短信或任意含金额文字）、手动输入。
- 账单列表：浏览、按分类/时间筛选、编辑、删除。
- 统计页：周/月总支出、分类占比、预算进度与超支提示。
- 剩余总额：初始总额录入 + 滚动展示。
- 设置（"我的"页）：DeepSeek API Key、预算设置、初始总额、分类查看。
- 首次使用引导：录入初始总额、提示配置 Key。

不覆盖：真实的 DeepSeek 调用、真实 OCR/语音识别、真实数据持久化、真实的 iOS 快捷指令与后台推送——demo 用假数据、模拟延时和顶部通知横幅来呈现"识别中→出结果/后台入账"的过程。

## 2. 交互原则

- **记账最短路径**：任何一种记账，从进入到入账完成不超过 3 步。
- **识别后直接入账、当场可改**：识别完就是一笔正式账，弹结果卡片让用户扫一眼、必要时当场改，关掉即完成；不做"待确认队列"。
- **状态始终可见**：识别这类异步操作必须有"识别中"的明确态；失败有明确提示和退路（转手动）。
- **破坏性操作有确认**：删除账单需二次确认。
- **金额醒目**：支出/收入用不同符号或颜色区分方向，金额是每个列表项的视觉重点。

### 4.2.1 快捷指令截图 · 后台入账 + 通知（主入口）

截图记账的主路径不在 App 界面内，而是用户在任意付款页触发 iOS 快捷指令把截图发给 Aubade，App 在**后台**识别并直接入账，只弹一条顶部通知：

```
① 收到快捷指令截图        ┌──────────────────────────┐
   （后台识别中…）        │ ⟳  收到快捷指令截图        │  ← loading 横幅
                         │    后台识别中…（本地→DS）  │
                         └──────────────────────────┘
② 识别成功、已入账        ┌──────────────────────────┐
   弹结果通知             │ 🌅 已记一笔 · 支出 ¥42.80  │  ← 点此打开结果卡片
                         │    🍜 食 · 瑞幸咖啡 · 查看  │
                         └──────────────────────────┘
③ 识别失败               ┌──────────────────────────┐
   （不误记账）           │ ⚠️ 这张截图没识别出金额     │  ← 点此转手动补录
                         │    点此打开 Aubade 补录     │  （原图已保留）
                         └──────────────────────────┘
```

- 通知不打断用户当前操作；点击成功通知打开结果卡片可当场改，点击失败通知转手动补录。
- 识别失败时**不生成账单**，避免脏数据。
- demo 里用底部演示控制台的「📷 模拟快捷指令截图」按钮触发这条路径（勾"模拟识别失败"可看失败通知）。

### 4.3 识别结果卡片（截图/语音/文本识别共用）

识别完成后从底部弹出，账单**此时已入账**，卡片用于"看一眼 + 可当场改"：

```
┌─────────────────────────────┐
│ ✓ 已记一笔                    │
├─────────────────────────────┤
│ 金额   [ -35.00      ]        │  ← 可编辑
│ 方向   ( 支出 ● / 收入 ○ )    │
│ 分类   [ 食 ▾ ]               │  ← 下拉可改
│ 时间   [ 7月10日 12:30 ]      │
│ 商户   [ 某餐厅 ]             │
│ 备注   [            ]         │
├─────────────────────────────┤
│  [ 删除这笔 ]      [ 完成 ]    │
└─────────────────────────────┘
```

- "完成"关闭卡片，保留修改。
- "删除这笔"= 撤销这次入账，二次确认。
- 原始输入（截图文字/语音文字/短信原文）折叠展示，供核对。

## 5. 状态与异常

| 场景 | 状态呈现 | 恢复路径 |
|---|---|---|
| 识别进行中 | 全屏/卡片"识别中…"loading，禁止重复提交 | 可取消返回 |
| 识别成功 | 弹结果卡片，账单已入账 | 当场改或完成 |
| 识别失败（金额没解析出） | 提示"没能识别出金额"，保留原始文本 | 一键转手动填写，原文带入备注 |
| 快捷指令截图 · 后台识别中 | 顶部 loading 通知横幅"收到快捷指令截图 · 后台识别中…" | 等待结果，不打断当前操作 |
| 快捷指令截图 · 成功入账 | 顶部通知"已记一笔 · 金额 · 分类"，账单已入账 | 点通知打开结果卡片当场改，不点也已记好 |
| 快捷指令截图 · 识别失败 | 顶部通知"没识别出金额"，**不生成账单**，原图保留 | 点通知打开 App 转手动补录 |
| 未配置 DeepSeek Key 就用识别功能 | 拦截提示"请先在『我的』配置 DeepSeek Key"；快捷指令后台触发时以通知形式提示 | 跳转"我的"填 Key；手动记账不受影响 |
| 剪贴板为空点"读取剪贴板" | 轻提示"剪贴板没有内容" | 手动粘贴 |
| 账单列表为空 | 空状态引导去记账 | — |
| 统计某天无账单（日档） | "这一天还没有账单" | 翻到别的日期/去记账 |
| 统计某期无支出（周/月/年档） | 趋势图显示"本期还没有支出"占位、占比区空态 | 翻到别的时间区间 |
| 统计时间导航翻到今天所在区间 | `›`（下一个）禁用置灰，不能翻到未来 | 只能往过去翻 |
| 点分类占比某一类 | 弹出该分类在当前时间区间的记录明细（含小计），点某笔进详情 | 关弹窗返回；改/删账单 |
| 未设预算 | 预算区显示"未设置预算，去设置" | 跳"我的" |
| 删除账单 | 二次确认弹窗 | 取消/确认 |
| 首次启动未录初始总额 | 剩余总额显示"—"，引导录入 | 去引导页/我的录入 |

## 6. 主要评审路径

打开 demo 默认就在「记账」页。建议你按这个顺序点一遍，边点边想"顺不顺手、缺不缺东西"：

1. **快捷指令截图记账**（新，重点体验）→ 点"截图识别"看说明卡 → 点"演示：模拟收到一张快捷指令截图" → 看顶部通知从"后台识别中"变成"已记一笔 ¥42.80"，记账页不被打断、最近记录立刻出现这笔 → 点通知打开结果卡片改一下。也可用底部控制台的「📷 模拟快捷指令截图」直接触发。
2. **文本识别记账**（最核心）→ 点"文本识别" → "读取剪贴板"带入示例 → 看识别中 → 看结果卡片 → 改一下分类 → 完成 → 切到账单页看到这笔 + 剩余总额变化。
3. **相册选图 / 语音记账** → 截图说明卡里点"从相册选一张图识别"，或点语音，走一遍识别中→结果卡片。
4. **手动记账** → 填一笔收入（工资）。
5. **账单筛选** → 按"食"筛、按"本周"筛。
6. **改/删账单** → 点一笔改分类；删一笔看确认弹窗。
7. **统计时间维度**（本次重点）→ 进"统计"，顶部切「日/周/月/年」，用「‹ ›」前后翻时间：切到「日」选某天看当天流水、切到「月」翻看上月、切到「年」看全年每月趋势；注意趋势横轴跟随粒度、`›` 不能翻到未来；在周/月/年档**点分类占比里的某一类**，看弹出的该分类明细（点某笔还能进详情改），看分类占比和**月预算超支标红**。
8. **设预算** → 去"我的"改预算 → 回统计看进度变化。
9. **异常** → 勾"模拟未配置 Key"后触发快捷指令/点识别看拦截提示；勾"模拟识别失败"后触发快捷指令，看失败通知且**没有误记账**。
10. **首次引导** → 点"重置数据"回到首次引导，录初始总额 → 落到记账页（空账本）。

## 7. 真实交互 demo

- 位置：`prototype/app/`，入口 `prototype/app/index.html`。
- 打开方式：见 `prototype/README.md`（双击 index.html 即可在浏览器打开，无需环境）。
- 技术：纯静态 HTML/CSS/JS，无构建、无依赖；用假数据 + setTimeout 模拟识别延时与结果。
- 定位：仅评审基线，非生产前端代码；真实 App 用 SwiftUI 实现。

## 8. 不做范围

- 不接真实 DeepSeek / OCR / 语音识别，识别结果为预置示例。
- 不做真实持久化（刷新页面数据重置）。
- 不做像素级视觉与动效打磨（低保真线框）。
- 不实现 SwiftUI/原生代码——那是技术基线与开发阶段的事。
- 不实现真实的 iOS 快捷指令、App Intents/分享扩展、系统通知——demo 用"控制台按钮触发 + 顶部横幅"来模拟这条路径；真实接入形态留待技术基线。
- 不覆盖多账户、iCloud 同步等 PRD 已排除的范围。
